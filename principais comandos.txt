GitHub Desktop

Após fazer alterações no projeto, o fluxo no GitHub Desktop é:
1. Ver as alterações

Abra o GitHub Desktop
Na esquerda aparece a lista de arquivos alterados

2. Fazer o Commit

Em baixo à esquerda escreva uma mensagem descrevendo o que fez
Ex: Layout responsivo mobile
Clique em Commit to main

3. Enviar para o GitHub

Clique em Push origin (botão azul no topo)

4. Atualizar o servidor

No terminal SSH do servidor:

cd /var/www/saproweb && git pull && php artisan view:clear && php artisan cache:clear
Pronto! Em um comando o servidor fica atualizado! 🚀⚖️

Crie o módulo de Publicações AASP no SAPRO com:

1. MIGRATION: tabela aasp_advogados (id, nome, codigo_aasp, chave_aasp, email, ativo)
   tabela aasp_publicacoes (id, codigo_aasp, data, jornal, numero_processo, titulo, texto, numero_publicacao, created_at)

2. COMPONENTE Livewire AaspPublicacoes com:
   - Cadastro de advogados com código AASP e chave individual
   - Busca manual por data (padrão = hoje)
   - Botão "Buscar Publicações" que chama a API:
     GET https://intimacaoapi.aasp.org.br/api/Associado/intimacao/json
     ?chave={chave_aasp}&data={data}&diferencial=false
   - Busca todos os advogados cadastrados em sequência
   - Salva resultados na tabela aasp_publicacoes
   - Exibe resultado em tabela com: data, advogado, jornal, processo, título
   - Botão "Gerar Word" que monta documento unificado com todas as publicações do dia
   - Botão "Enviar por E-mail" que envia o documento para lista de e-mails configurada

3. CONFIG: tabela aasp_config (emails_destino, horario_rotina, ativo)

4. ROTA: /aasp-publicacoes

5. MENU: adicionar item "📰 Publicações AASP" visível só para admin e advogado

Analise o projeto existente antes de criar. Use Http::get() do Laravel para chamar a API.
